Network fundamentals

BC2 consensus at a glance

Core parameters published on the official Bitcoin II launch site. Mining-ready, halving-aligned, and supported by SHA-256 hardware fleets.

  • Consensus: Proof-of-Work secured by the SHA-256 hashing algorithm.
  • Max supply: 21,000,000 BC2 with the same halving cadence of every 210,000 blocks.
  • Difficulty retarget: Adjusts every 2,016 blocks so block times remain consistent as hashrate changes.
  • Fresh genesis: Bitcoin II starts from a brand new blockchain, giving miners an even playing field.
  • Mining cadence: Launch difficulty reset to 1 for fairness, while today's security relies on SHA-256 ASIC hash power.
Keeping blocks lean

Bitcoin II Core retains the 80-byte OP_RETURN ceiling so transaction data stays prioritized over arbitrary payloads.

Future consensus work is focused on reducing non-transactional storage vectors altogether—follow progress in the Daily Network Report.
