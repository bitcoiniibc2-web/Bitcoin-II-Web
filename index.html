What Bitcoin II offers

  • SHA-256 Proof-of-Work: Same hashing algorithm and security assumptions as Bitcoin Core.
  • Fixed 21,000,000 BC2 supply: Mining subsidy halves every 210,000 blocks, echoing Bitcoin-II.org guidance.
  • Difficulty retargeting: Adjusts every 2,016 blocks so block times stay close to ten minutes.
  • Difficulty restart at 1: Launch conditions were tuned for CPUs, matching the official mining primer.
  • Fresh genesis block: A brand new chain so miners, holders, and builders start on equal footing.

Network fundamentals

Core parameters published on the official Bitcoin II launch site.

  • Consensus Proof-of-Work secured by the SHA-256 hashing algorithm.
  • Max supply 21,000,000 BC2 with the same halving cadence of every 210,000 blocks.
  • Difficulty retarget Adjusts every 2,016 blocks so block times remain consistent as hashrate changes.
  • Fresh genesis Bitcoin II starts from a brand new blockchain, giving miners an even playing field.
  • Mining accessibility Launch difficulty was reset to 1, making CPU mining viable for early supporters.

Looking to mine? Recommended tooling includes Pooler's cpuminer with community pools listed below.

Seed a new node

Until the DNS seeds propagate, manually add the community node from the Bitcoin-II.org guide:

addnode bitcoinII.ddns.net:8338 add

Run the command in BitcoinII-Qt's console or via bitcoin-cli to connect instantly.
